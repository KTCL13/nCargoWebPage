¿Qué dirección tomamos para unificar?

A

Tokens primero

Login/forgot-password adoptan las CSS variables y clases del proyecto (var(--color-nc-*), btn-primary, form-input). Sin cambiar el look visual — mismos colores, mismas formas — solo se reemplaza el código hardcodeado. Cambio mínimo, máximo impacto en consistencia.

B

Componentes compartidos

Crear componentes reutilizables (AuthInput, AuthButton, AuthCard) usados por login, forgot-password y reset-password. Los modales del dashboard también usan ModalShell unificado con cabecera + botón Cancelar estándar.

C

Tokens + Componentes (completo)

Ambas anteriores: reemplazar hardcodes AND crear los componentes compartidos. Login/forgot/reset usan los mismos tokens y un AuthFormCard común. Dashboard: ModalShell unificado para todos los modales con Cancelar estándar.

Enfoques de implementación

Tres formas de lograr tokens + componentes + navegación móvil unificada

A

Componentes React compartidos ✦ Recomendado

Crear 3 piezas nuevas en src/components/ que unifican todo:

AuthFormCard — layout de dos columnas (imagen izq + card der) reutilizado por login, forgot-password y reset-password. Cada página solo aporta su formulario como children.
ModalShell — envuelve todos los modales: cabecera (título + subtítulo + ✕), área de contenido, y pie estandarizado con botón Cancelar + slot para acción primaria.
useDirtyForm(initial) — hook que compara estado actual vs inicial. Devuelve isDirty y reset(). ModalShell lo usa para decidir si mostrar "¿Descartar cambios?".

Pros

  • Un solo lugar para cambiar el estilo de todos los modales
  • Dirty-check reutilizable en cualquier modal futuro
  • Auth pages quedan muy simples

Cons

  • Más refactoring inicial (tocar todos los modales)
B

Solo CSS clases + hook

Sin nuevos componentes React. Agregar .auth-card, .modal-header, .modal-footer a globals.css. Cada página/modal los adopta añadiendo clases. El hook useDirtyForm sí se crea.

Pros

  • Menos archivos nuevos
  • No hay que tocar las interfaces de los componentes existentes

Cons

  • El markup puede divergir otra vez (no hay contrato)
  • El Cancelar hay que agregarlo manualmente en cada modal sin garantía de consistencia
C

ModalShell únicamente (sin AuthFormCard)

Crear solo ModalShell + useDirtyForm. Las auth pages (login/forgot/reset) solo se limpian de tokens hardcodeados pero no adoptan un componente común — siguen siendo páginas independientes.

Pros

  • Menos cambios en auth pages (riesgo de regresión bajo)
  • ModalShell aun así unifica todos los modales del dashboard

Cons

  • Si se agrega una 4ª página de auth, hay que copiar la estructura otra vez
  • Login y forgot-password siguen siendo idénticas en estructura pero sin contrato
En todos los enfoques se elimina BottomNav. El DashboardLayout quita el import, el JSX y el padding inferior. En móvil queda solo el hamburger → Sidebar.
